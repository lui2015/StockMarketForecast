## 1. 项目概述

### 1.1 背景
用户（含个人投资者与 AI 程序）需要一套系统，用于**每日提交对股票市场的涨跌预测**，并在**次日自动校验预测准确性**，长期沉淀预测记录并统计准确率。

覆盖四类预测标的：
- A 股大盘（如上证指数、深证成指、创业板指）
- 港股大盘（如恒生指数、恒生科技指数）
- A 股个股
- 港股个股

### 1.2 目标
1. 支持用户/AI 每日提交结构化预测（含涨跌方向 + 预测逻辑）。
2. 系统次日自动拉取行情并校验，判定命中/未命中。
3. 提供个人及全局的预测准确率、累积准确率统计与可视化。
4. 移动端优先的页面体验。
5. 对外开放标准 HTTP 接口，供 AI Agent 直接调用提交预测。

### 1.3 核心术语
| 术语 | 说明 |
|------|------|
| 预测单 | 一次提交的预测记录，含标的、方向、逻辑、目标日期 |
| 校验 | 次日对比实际行情，判定预测是否命中 |
| 命中 | 预测方向与实际涨跌方向一致 |
| 准确率 | 命中数 / 总已校验预测数 |
| 累积准确率 | 全周期累计命中数 / 全周期已校验总数 |

## 3. 功能需求

### 3.1 预测提交（PRD-F001）
- 用户选择**预测标的类型**（A股大盘 / 港股大盘 / A股个股 / 港股个股）。
- 选择**具体标的**（大盘选指数，个股输入/搜索股票代码）。
- 选择**预测方向**：涨 / 跌（可选扩展：平）。
- 填写**预测逻辑**（自由文本，说明判断依据，必填）。
- 指定**目标交易日**（默认下一个交易日）。
- 提交后生成一条预测单，状态为「待校验」。
- 约束：
  - 同一用户、同一标的、同一目标交易日，**仅允许一条有效预测**：重复提交将**覆盖**为最新一条（更新涨跌方向与预测逻辑），而非拒绝。
  - 校验弹窗（管理员视角）按「标的 + 目标交易日」去重，**跨用户仅展示每个标的最新一条**预测，避免同一标的重复校验。
  - 提交时间限制：需在目标交易日开盘前（可配置截止时间，如 09:00/09:30）。

### 3.2 自动校验（PRD-F002）
- 每日收盘后（或定时任务，如 16:30）触发校验任务：
  1. 拉取目标交易日各标的最新收盘价。
  2. 对比预测方向与实际涨跌（实际涨跌幅 = (收盘价 - 前收盘价) / 前收盘价）。
  3. 判定命中：预测「涨」且实际 >0 → 命中；预测「跌」且实际 <0 → 命中；其余未命中。
  4. 更新预测单状态为「已校验」，记录实际涨跌幅、是否命中。
- 行情数据源：优先第三方行情 API（如 Tushare、新浪/腾讯财经、东方财富、AKShare）。需配置 fallback 源。
- 行情兜底：当外部行情源不可用时（如接口限流/封锁），系统使用权威历史收盘数据（如 FRED NASDAQCOM 纳斯达克综合指数）作**静态种子**补全历史交易日数据，保证统计完整性（实时源恢复后仍以实时数据为准）。
- 校验失败（行情缺失）的预测单标记为「校验异常」，支持管理员手动补录。

### 3.3 统计与展示（PRD-F003）
- **个人维度**：
  - 总体准确率、累积准确率
  - 按标的类型拆分（A股大盘 / 港股大盘 / A股个股 / 港股个股）的准确率
  - 近期命中趋势（折线图，按时间）
  - 连胜 / 连败计数
- **全局维度（可选）**：
  - 所有用户综合准确率排行（Leaderboard）
- 统计为实时聚合（或按日缓存），避免每次全表扫描。

### 3.4 历史与明细（PRD-F004）
- 预测列表：分页展示个人的历史预测，含标的、方向、逻辑、目标日、实际结果、命中状态。
- 支持按标的类型、时间范围、命中状态、提交来源筛选。
- 预测详情：完整查看单条预测与其校验结果。
- 编辑与删除：
  - 列表项提供「编辑」按钮，本人或管理员可修改涨跌方向与预测逻辑；**若方向发生变更，系统自动将状态重置为「待校验」并清空原校验结果**（实际涨跌幅、是否命中）。
  - 编辑弹层支持「删除该预测单」（不可逆，需二次确认），删除时一并清理关联的预测逻辑文件。

### 3.5 移动端页面（PRD-F005）
- 移动端优先（Mobile First）响应式设计。
- 页面包括：
  - 首页/仪表盘：今日待提交、近期准确率概览
  - 提交页：分步选择标的 → 方向 → 逻辑
  - 统计页：图表展示
  - 历史页：列表与筛选
- 适配主流手机分辨率（375/414 等），支持手势操作、下拉刷新。
- 移动端适配要点（已落地）：
  - 顶部 Tab 横向滑动，文字不换行。
  - 成功率 / 准确率指标恢复为**三列并列**，不占满整屏。
  - 今日三大盘（沪深 / 港股 / 美股）**三列并列**展示，卡片内紧凑排版。
  - 预测日历筛选条横向滑动，日期格子紧凑。
  - 历史页筛选框换行铺满：市场分段占整行、状态/方式各占 50%、日期范围整行；**个股下拉与首页预测日历共用同一标的列表，保持一致**。

### 3.6 开放接口（AI 调用）（PRD-F006）
- 提供标准 RESTful API，供 AI Agent 提交预测，无需登录页面。
- 认证方式：API Key（每个调用方分配独立 key，Header 携带）。
- 接口能力：
  - 提交预测
  - 查询个人统计
  - 查询历史预测
- 限流：按 key 限流（如每分钟 N 次），防止滥用。

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 兼容性 | 移动端 Safari/Chrome（iOS/Android），主流分辨率 |
| 性能 | 提交接口 P95 < 500ms；统计页首屏 < 2s |
| 安全 | API Key 加密存储；预测接口鉴权；用户输入做转义防 XSS；参数校验防注入 |
| 可用性 | 定时校验任务具备失败重试；行情源具备 fallback |
| 可观测 | 任务执行日志、接口访问日志、异常告警 |
| 数据留存 | 预测与校验记录长期保存，支持导出 |

## 5. 数据模型（草案）

### 5.1 预测单 Prediction
```
id              PK
user_id         提交用户/调用方标识
market          枚举: A_INDEX / HK_INDEX / A_STOCK / HK_STOCK
symbol          标的代码（大盘为指数代码）
symbol_name     标的名称
target_date     目标交易日
direction       枚举: UP / DOWN   (可选 FLAT)
reason          预测逻辑文本
status          枚举: PENDING / VERIFIED / ERROR
actual_change   实际涨跌幅(校验后写入)
is_hit          是否命中(校验后写入)
created_at
updated_at
unique(user_id, symbol, target_date)
```

### 5.2 用户 / 调用方 Account
```
id
name
api_key         (加密存储)
type            枚举: USER / AI
rate_limit      限流配置
created_at
```

### 5.3 行情快照 Quote（可选，用于溯源）
```
symbol
trade_date
close_price
prev_close
change_pct
source
```

## 6. 接口设计（草案）

### 6.1 提交预测
```
POST /api/v1/predictions
Headers: Authorization: Bearer <API_KEY>
Body:
{
  "market": "A_INDEX",
  "symbol": "000001.SH",
  "target_date": "2026-07-22",
  "direction": "UP",
  "reason": "政策利好+资金面宽松"
}
Resp 200:
{ "code": 0, "data": { "id": "...", "status": "PENDING" } }
```

### 6.2 查询统计
```
GET /api/v1/stats?market=A_INDEX
Headers: Authorization: Bearer <API_KEY>
Resp:
{
  "code": 0,
  "data": {
    "total": 120, "hits": 78, "accuracy": 0.65,
    "by_market": { "A_INDEX": 0.70, "HK_STOCK": 0.60 }
  }
}
```

### 6.3 查询历史
```
GET /api/v1/predictions?page=1&size=20&status=VERIFIED
Headers: Authorization: Bearer <API_KEY>
```

### 6.4 校验任务（内部）
```
定时(每日收盘后): 拉取行情 → 更新 PENDING 预测单 → 重算统计缓存
```

## 7. 技术架构建议
- **前端**：移动端优先的 SPA（如 React/Vue + Tailwind），图表用 ECharts/Chart.js。
- **后端**：RESTful 服务（Node/Java/Python 均可），负责预测、校验、统计。
- **定时任务**：Cron / 消息队列 触发每日校验。
- **数据库**：关系型（PostgreSQL/MySQL）存预测与账户；行情可缓存。
- **行情源**：Tushare / AKShare / 财经公开接口，需多源容灾。
- **部署**：容器化（Docker），可参考既有 `investmentQuotes` 项目的 Lighthouse 部署方式。

---

## 8. 验收标准
1. 用户可在移动端完成一次完整预测提交流程。
2. 次日系统自动校验且准确率/累积准确率正确更新。
3. 统计页图表在手机端正常展示。
4. AI 调用方使用 API Key 能成功提交预测并查询统计。
5. 重复标的+目标日仅保留一条有效预测。
6. 行情缺失时预测单进入异常态，可人工补录。

## 9. 后续可扩展

- 多空点位/目标价预测（非仅涨跌）。

- 推送通知：每日提交提醒、校验结果推送。
